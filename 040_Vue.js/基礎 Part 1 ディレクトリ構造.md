### プロジェクトの主要なフォルダ

### 1. `src` フォルダ

- **概要**: 開発者がコードを書く、プロジェクトの**心臓部**です。
- **内容**: Vueコンポーネント（`.vue`ファイル）、JavaScriptファイル、[[ULCOD CSS|CSS]]ファイルなど、アプリケーションの機能やデザインを定義するすべてのコードが含まれます。

### 2. `public` フォルダ

- **概要**: [[コンパイルとビルド|ビルド]]プロセスによって処理されない、静的なファイルを格納する場所です。
- **内容**: `index.html`（テンプレート）、画像、フォント、`robots.txt`、`manifest.json`など、[[コンパイルとビルド|ビルド]]ツールがそのままコピーするファイルが含まれます。
- **特徴**: 開発時も[[コンパイルとビルド|ビルド]]後も、ファイルへのパスは変わりません。

### 3. `dist` フォルダ

- **概要**: [[コンパイルとビルド|ビルド]]コマンド (`npm run build`) の実行によって生成される、**本番環境用の成果物**が格納されるフォルダです。
- **内容**: `src` フォルダの内容が、Webブラウザで効率的に実行できるように最適化された状態で出力されます。
- **特徴**:
    - **バンドル**: 複数のファイルが一つにまとめられます。
    - **難読化**: [[AI激動期における「定数」思考|変数]]名などが短縮され、人間が読みにくく、かつファイルサイズが小さくなります。
    - **圧縮**: 不要な空白や[[改行コード|改行]]が削除されます。

###  主要なコマンド

- `**npm create @latest**`: 特定のフレームワーク（例: `vue`）の**ひな形を対話形式で作成**するコマンド。複雑な設定をすることなく、すぐに開発を始められます。
- `**npm install**`: `package.json` に記述された必要な**パッケージをインストール**するコマンドです。

---

###  package.[[設定ファイル言語|json]] の役割と構成

`package.json` は、プロジェクトの**依存関係（パッケージ）と実行スクリプト**を管理する[[設定ファイル言語|設定ファイル]]です。

### `dependencies` と `devDependencies`

このセクションは、プロジェクトが動作するために必要なパッケージをリスト化します。

- `**dependencies**`: アプリケーションを**本番環境で動かす**ために必須なパッケージがリスト化されます。これには、ユーザーが直接利用する機能（UIライブラリやデータ通信ライブラリなど）が含まれます。
- `**devDependencies**`: **開発時のみ**に必要なパッケージがリスト化されます。[[コンパイルとビルド|ビルド]]ツール（[[Vite HMR|Vite]]など）、コード品質チェックツール（[[ESLint]]）、テストツールなどがこれに該当します。本番環境には含まれません。

### scripts

`scripts` セクションは、**カスタムコマンドを定義する場所**です。

- `"scripts": { "dev": "vite" }`のように**キーと値**で構成され、`npm run dev`と実行すると`vite`コマンドが実行されます。
- この値は、プロジェクトの**`node_modules/.bin`フォルダ**にある実行ファイルと紐づいています。`npm run` コマンドは、このパスを自動的に認識してコマンドを実行してくれます。これにより、複雑なコマンドを簡潔な名前に置き換えることができます。

### 開発環境とツールの関係

- **[[Vite HMR|Vite]]**: 次世代のフロントエンド開発ツール。Vue.jsと相性が良く、開発サーバーの起動やコードの自動変換を高速に行います。
- **Node.js**: JavaScriptをブラウザの外で実行するための実行環境。[[npm]]コマンドや開発サーバーの動作に必要です。
- **VS Code 推奨拡張機能**:
    - **Vue. Volar**: `.vue` ファイルを正しく認識し、構文のハイライトや補完を可能にします。
    - **[[ESLint]]**: コード品質をチェックし、問題があれば警告してくれます。
    - **[[Prettier]]**: コードのフォーマットを自動的に統一し、チーム開発でのコードスタイルを維持します。
