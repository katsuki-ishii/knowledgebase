1. トリガーの大分類

関数が実行されるきっかけは大きく分けて2種類存在する。

1. 時間によるトリガー（[[イベント駆動と時間駆動|タイマードリブン]]）
2. 出来事によるトリガー（[[イベント駆動と時間駆動|イベントドリブン]]）

---

### 2. [[イベント駆動と時間駆動|タイマードリブン]]（[[イベント駆動と時間駆動|Time-driven]]）

時間の経過をトリガーとして関数を実行する仕組み。

### 種類

3. **スケジュール型（Scheduled）**
特定の時刻や日付に実行される。
例：毎日9時に実行、毎週月曜にレポートを生成する。
```javascript
// 毎日9時に実行（Node.js）
import cron from 'node-cron';
cron.schedule('0 9 * * *', () => {
  sendDailyReport();
});

```
4. **インターバル型（Interval）**
一定間隔ごとに繰り返し実行される。
例：5分ごとに状態をチェック、10秒ごとにUIを更新する。
```javascript
// 5分ごとに実行
setInterval(checkForUpdates, 5 * 60 * 1000);

```

---

### 3. [[イベント駆動と時間駆動|イベントドリブン]]（[[イベント駆動と時間駆動|Event-driven]]）

何かの出来事をトリガーにして関数を実行する仕組み。プログラムが「待機状態」にあり、出来事が発生した瞬間に関数が呼ばれる。

### 主な分類

5. **ユーザー[[基礎 Part 14 イベントオブジェクト|イベント]]（UI操作）**
クリックや入力など、ユーザーの操作に反応する。
```html
<button @click="saveData">保存</button>

```
6. **データ[[基礎 Part 14 イベントオブジェクト|イベント]]（[[基礎 Part 19 リアクティブエフェクト|リアクティブ]]）**
データの状態変化を監視して反応する。
```javascript
watch(userAge, (newAge) => {
  updateMilestones(newAge);
});

```
7. **システム[[基礎 Part 14 イベントオブジェクト|イベント]]（通信・非同期処理）**
API応答や[[Web Socket|WebSocket]]メッセージの受信などに反応する。
```javascript
socket.on('newMessage', displayMessage);

```
8. **内部[[基礎 Part 14 イベントオブジェクト|イベント]]（ランタイム・OS）**
プロセスやファイルの変更など、システム内部の出来事に反応する。
```javascript
process.on('exit', cleanup);
fs.watchFile('data.txt', reload);

```
9. **カスタム[[基礎 Part 14 イベントオブジェクト|イベント]]（開発者定義）**
開発者が独自に定義した[[基礎 Part 14 イベントオブジェクト|イベント]]を発火・監視する。
```javascript
const event = new CustomEvent('goalAchieved', { detail: 42 });
window.dispatchEvent(event);

```

---

### 4. [[イベント駆動と時間駆動|イベントドリブン]]と[[イベント駆動と時間駆動|タイマードリブン]]の違い

| 種類 | トリガー | 特徴 | 主な用途 |
| --- | --- | --- | --- |
| [[イベント駆動と時間駆動|イベントドリブン]] | 出来事の発生 | 予測できないタイミングで反応する | UI操作、通信、状態変化 |
| [[イベント駆動と時間駆動|タイマードリブン]] | 時間の経過 | 予測可能なタイミングで実行する | スケジュール、定期更新 |

---

### 5. 補足

- [[イベント駆動と時間駆動|イベントドリブン]]は「外部からの刺激」に反応する仕組み。
- [[イベント駆動と時間駆動|タイマードリブン]]は「自分で決めたリズム」で動く仕組み。
- 多くのアプリケーションはこの2種類の組み合わせで構成されている。
