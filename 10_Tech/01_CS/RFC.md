# 1. [[RFC]]とは

- **正式名称**: Request for Comments（コメント募集）
- **概要**: インターネットや関連技術の仕様・標準を定める文書
- **発行主体**: 主にIETF（Internet Engineering Task Force）
- **特徴**
    - 番号で管理（例: [[RFC]] 3986）
    - 誰でも閲覧・参加可能
    - オープンで無料
    - 必要に応じて更新・廃止あり
- **例**:
    - [[RFC]] 3986: [[URI・URL・URN|URI]]の構文
    - [[RFC]] 791: IPv4
    - [[RFC]] 9110: HTTP/1.1
    - [[RFC]] 8446: TLS 1.3

---

## 2. 名前の由来

- 1969年ARPANET時代、仕様案やアイデアを共有するために
「意見を聞かせてほしい」という意味で“Request for Comments”と命名
- 初期はカジュアルな研究メモ → 現在は事実上の標準仕様書
- 名前は歴史的経緯とオープン性を象徴して残っている

---

## 3. [[RFC]]の対象範囲

- **ネットワーク基盤**
    - IPv4（[[RFC]] 791）
    - IPv6（[[RFC]] 8200）
    - TCP（[[RFC]] 793）
    - UDP（[[RFC]] 768）
- **アプリケーション層**
    - HTTP（[[RFC]] 9110）
    - SMTP（[[RFC]] 5321）
    - DNS（[[RFC]] 1034, 1035）
- **セキュリティ**
    - TLS 1.3（[[RFC]] 8446）
    - [[OAuth]] 2.0（[[RFC]] 6749）
    - JWT（[[RFC]] 7519）
- **データ形式**
    - [[URI・URL・URN|URI]]（[[RFC]] 3986）
    - [[設定ファイル言語|JSON]]（[[RFC]] 8259）
- **ジョーク[[RFC]]**
    - [[RFC]] 1149（ハトでIPパケットを配送）

## 5. ISOとの違い

| 項目 | [[RFC]]（IETF） | ISO |
| --- | --- | --- |
| 対象 | インターネット技術 | 全産業分野 |
| 主体 | IETF（オープンコミュニティ） | 各国代表機関による組織 |
| 決定方法 | オープンな議論で合意形成 | 各国の投票で承認 |
| 公開性 | 無料 | 有償（PDF販売） |
| スピード | 数か月〜数年 | 数年〜十年以上 |
| 採用 | 現場で即実装 | 法規・契約でも使用 |

## 6. 信頼性と引用

- **信頼性の理由**
    - 国際的に認知された標準化機関が策定
    - 世界中の専門家レビュー
    - 永続的な番号管理
- **引用可否**
    - 論文・企業報告書で引用可能
    - 最新版の確認が必要
    - 公式[[URI・URL・URN|URL]]を使用（例: [https://www.rfc-editor.org/rfc/rfc3986）](https://www.rfc-editor.org/rfc/rfc3986%EF%BC%89)

### 引用例（APA形式）

```plain text
Berners-Lee, T., Fielding, R., & Masinter, L. (2005).
Uniform Resource Identifier (URI): Generic Syntax (RFC 3986).
RFC Editor. https://www.rfc-editor.org/rfc/rfc3986

```

## 7. ティム・バーナーズ＝リー

- **概要**: Web（WWW）の発明者、W3C創設者
- **功績**
    - HTML、HTTP、[[URI・URL・URN|URI]]の基礎設計
    - 世界初のWebサーバとブラウザ開発
    - ナイト爵位授与（Sir）
- **現在**
    - 分散型Web（Solidプロジェクト）推進
    - オープンWebの擁護活動

## まとめ

[[RFC]]はインターネットの「設計図・憲法」であり、世界的に信頼される一次資料。

ISOとはプロセスや対象領域が異なるが、どちらも国際標準としての価値を持つ。

ティム・バーナーズ＝リーはその一部を直接作った歴史的かつ現役の人物。
