# [[基礎 Part 14 スプレット構文|スプレッド構文]]まとめ

## 1. [[基礎 Part 14 スプレット構文|スプレッド構文]]とは

[[基礎 Part 14 スプレット構文|スプレッド構文]]は `...` を使って、配列やオブジェクトの「中身を広げる」ための JavaScript の文法である。箱そのものを渡すのではなく、中身だけを取り出して並べるイメージで捉えると理解しやすい。

## 2. 基本的なイメージ図

```plain text
[1, 2, 3] という箱
    │
    └── ... を使う
            │
            ▼
1, 2, 3 がむき出しになる

```

```plain text
{ a: 1, b: 2 } という箱
    │
    └── ... を使う
            │
            ▼
a: 1, b: 2 がむき出しになる

```

## 3. 配列での使用例

### 3-1 配列のコピー

```javascript
const a = [1, 2, 3];
const b = [...a];

```

中身だけ展開して新しい配列に入れているため、安全にコピーできる。

### 3-2 配列の結合

```javascript
const a = [1, 2];
const b = [3, 4];
const c = [...a, ...b]; // [1, 2, 3, 4]

```

箱同士をつなげるのではなく、中身を取り出して並べているだけである。

## 4. オブジェクトでの使用例

### 4-1 オブジェクトのコピー

```javascript
const user = { name: "Alice", age: 20 };
const copy = { ...user };

```

プロパティを新しいオブジェクトに入れ直している。

### 4-2 上書きを含む結合

```javascript
const user = { name: "Alice", age: 20 };
const updated = { ...user, age: 21 };

```

右側に書いた値が優先される。

図解：

```plain text
展開: name: "Alice", age: 20
             │
             ▼
{ name: "Alice", age: 21 }  // age が後から上書きされる

```

## 5. [[基礎 Part 14 スプレット構文|スプレッド構文]]を理解するための中心イメージ

### 「箱を渡すのではなく、中身をぶちまけて渡す」

このイメージで覚えると、コピー・結合・上書きの挙動が一貫して理解できる。

## 6. 注意点：浅いコピーであること

[[基礎 Part 14 スプレット構文|スプレッド構文]]は 1 階層目だけ新しく作る。入れ子のデータはコピーされず参照が共有される。

```javascript
const a = { nested: { x: 1 } };
const b = { ...a };
b.nested.x = 2;

```

この場合、`a.nested.x` も 2 に変わる。

図解：

```plain text
a.nested ----→ { x: 1 }
                 ▲
                 │ 共有されている
b.nested --------┘

```

## 7. まとめ

- [[基礎 Part 14 スプレット構文|スプレッド構文]]は `...` で中身を展開する文法
- 配列・オブジェクトのコピー、結合、上書きで使用する
- あくまで浅いコピーである
- 中身を取り出して新しい器に盛り直すイメージで理解するとよい
