##  1. 用語の違い

| 用語 | 例 | 意味 |
| --- | --- | --- |
| 宣言 | `let x;` | 変数（箱）を作るだけ |
| 代入 | `x = 10;` | 変数に値を入れる（上書き含む） |
| 宣言＋代入 | `let x = 10;` | 変数を作って値を同時に入れる |
| 再代入 | `x = 20;` | 変数に別の値を入れ直す |

## 2. キーワード別の特徴

| キーワード   | 再宣言 | 再代入 | 主な用途       |
| ------- | --- | --- | ---------- |
| `let`   | ❌   | ✅   | 一般的な変数     |
| `const` | ❌   | ❌   | 定数（値変更禁止）  |
| `var`   | ✅   | ✅   | 古い書き方（非推奨） |

### ポイント

- `let`や`const`は**ブロックスコープ**を持つ。
- `var`は**関数スコープ**で、再宣言可能なためバグの温床になりやすい。

## 3. よくあるエラーの原因

### 🔴 例：再宣言によるエラー

```plain text
let val = init;
for (...) {
    let val = fn(val, nums[i]); // ❌ valを再宣言しているためエラー
}

```

### ✅ 修正例：再代入でOK

```plain text
let val = init;
for (...) {
    val = fn(val, nums[i]); // ✅ 既存のvalに再代入している
}

```

## 4. 練習問題

次のうちエラーになるのはどれ？

```plain text
let a = 5;
let a = 10; // ① ❌ 同じスコープ内で再宣言

const b = 20;
b = 25;     // ② ❌ constは再代入できない

var c = 3;
var c = 4;  // ③ ✅ ただし非推奨

let d;
d = 15;     // ④ ✅ 問題なし

```

## 5. まとめ

- **宣言**：変数を作る → `let`, `const`, `var`
- **代入**：値を入れる・更新する → `=`
- **再代入**：変数の中身を上書きする（宣言不要）
- **再宣言は禁止**：同じスコープでの再宣言は`let`や`const`ではNG

この3つの区別をしっかり理解することで、エラーを減らし、読みやすいコードが書けるようになる！
