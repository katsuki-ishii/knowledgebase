# [[O記法（ビッグオー記法）]]

## 何を表すのか

「データ数nが増えたとき、処理回数がどう増えるか」を表す記号。

具体的な処理時間（秒）ではなく、**増え方のパターン**を表す。

---

## 具体例で理解する

### O(n) の場合 — [[線形探索]]

```
データ: [3, 7, 1, 9, 4]  → 最大5回比較
データ: [3, 7, 1, ..., 4]  → 100件なら最大100回比較
```

データが増えた分だけ、処理回数も増える。これが O(n)。

### O(1) の場合

```
データが5件でも100件でも、常に1回で終わる
```

[[線形探索]]で先頭に答えがあれば、1回比較した瞬間に終わる。これが O(1)。

---

## 図解

```
処理回数
|
|              O(n²)
|           /
|         /
|       /     O(n)
|     /    __/
|   /  __/
|  /__/_________ O(1)
+-------------------→ データ数(n)
```

O(1) はどこまでいっても横一直線。  
O(n) はデータに比例して増える。  
O(n²) はデータが増えると爆発的に増える。

---

## なぜ「n/2回」でも O(n) と書くのか

[[線形探索]]の平均は n/2 回だが、O(n/2) とは書かない。

**[[AI激動期における「定数」思考|定数]]倍の違いは無視する**というルールがあるからだ。

100万件 vs 200万件のとき、「n/2かnか」の差より「nかn²か」の差の方がはるかに大きい。スケールが大きくなったときの本質的な差だけを見たい、という発想。

## 最良・平均・最悪の3ケース

同じアルゴリズムでも、状況によって処理回数が変わる。

[[線形探索]]で 9 を探す場合：

| ケース | 状況             | [[O記法（ビッグオー記法）]] |
| --- | -------------- | ------------ |
| 最良  | 先頭にある          | O(1)         |
| 平均  | 真ん中あたりにある      | O(n)         |
| 最悪  | 末尾にある、または存在しない | O(n)         |

試験では「最悪」計算量が問われることが多い。
