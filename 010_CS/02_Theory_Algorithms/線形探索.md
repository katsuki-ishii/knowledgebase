# [[線形探索]]

## 1. 概要

[[線形探索]]（[[線形探索|Linear Search]]）とは、データ集合の先頭から順番に要素を調べ、目的の値を探す最も基本的な探索アルゴリズムである。

特別な前処理や並び替えは不要であり、単純な構造のデータに対してそのまま適用できる。

---

## 2. 原理

配列やリストの先頭から1つずつ順番に比較していく。

例えば次の配列があるとする。

[3, 8, 2, 10, 5]

「10」を探す場合の流れは以下の通りである。

1. 3 と比較 → 不一致
2. 8 と比較 → 不一致
3. 2 と比較 → 不一致
4. 10 と比較 → 一致（探索終了）

このように、見つかるまで順番に比較し続ける。

---

## 3. 名前の由来

「[[線形探索|線形]]（linear）」とは、データ数の増加に対して処理回数が比例して増える性質を指す。

データ数を n とすると、最大で n 回比較する必要がある。つまり、データが2倍になれば処理回数もおおよそ2倍になる。この増え方が一次関数（直線的な増加）に相当するため、「[[線形探索]]」と呼ばれる。

---

## 4. 計算量

データ数を n とすると、時間計算量は [[O記法（ビッグオー記法）|O(n)]] である。

### 最悪の場合

目的の値が存在しない、または最後に存在する場合。
→ n 回比較する。

### 最良の場合

最初の要素が目的の値である場合。
→ 1 回で終了する。

データ数が増えると比較回数も比例して増えるため、「[[線形探索|線形]]」と呼ばれる。

---

## 5. 具体例（Python）

```python
def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1

numbers = [3, 8, 2, 10, 5]
print(linear_search(numbers, 10))  # 3
```

この関数は、見つかった場合はそのインデックスを返し、存在しない場合は -1 を返す。

---

## 6. [[基礎 Part 20 標準メソッド|標準メソッド]]との関係

多くのプログラミング言語では、[[線形探索]]は[[基礎 Part 20 標準メソッド|標準メソッド]]として実装されている。

例（Python）

```python
10 in [3, 8, 2, 10, 5]
[3, 8, 2, 10, 5].index(10)
```

これらは内部的に先頭から順番に比較している。

---

## 7. メリットとデメリット

### メリット

* 実装が非常に簡単
* データが整列されていなくても使える
* 小規模データでは十分実用的

### デメリット

* データ数が増えると遅くなる
* 毎回最初から探索する必要がある

---

## 8. 使用場面

* データ数が少ない場合
* 一時的なリストの中から値を探す場合
* ソートされていないデータを扱う場合

大量データや高速な存在確認が必要な場合は、ハッシュ構造（Setや[[基礎 Part 6 Map関数の自作|Map]]）や二分探索など別の方法を検討する必要がある。

---

## 9. まとめ

[[線形探索]]は、最も基本的な探索アルゴリズムであり、先頭から順番に調べるという単純な原理に基づく。

時間計算量は [[O記法（ビッグオー記法）|O(n)]] であり、データ数に比例して処理時間が増加する。

アルゴリズム理解の出発点として重要な概念である。
