## [[基礎 Part 5セミコロン・パース|セミコロン]]の役割

- JavaScriptでは、**文（statement）の区切り**として[[基礎 Part 5セミコロン・パース|セミコロン]]（;）を使う
- *自動[[基礎 Part 5セミコロン・パース|セミコロン]]挿入（ASI）**のおかげで省略できる場合もあるが、思わぬバグが起きることも

### ● 基本ルール

- **変数[[基礎 Part 8 宣言・代入・再代入|宣言]]、[[基礎 Part 8 宣言・代入・再代入|代入]]、return、式の終了などには基本[[基礎 Part 5セミコロン・パース|セミコロン]]をつける**
- **if, for, while, function[[基礎 Part 8 宣言・代入・再代入|宣言]]などのカッコの後には不要**
- **オブジェクトや配列の「中身」には[[基礎 Part 5セミコロン・パース|セミコロン]]不要（カンマ区切り）**

### ● 付けないとバグる例

```javascript
let a = 1
let b = 2
[a, b].forEach(console.log)
// → JSは「let b = 2[a, b].forEach...」と解釈しエラー

```

### ● return文の罠

```javascript
function foo() {
  return
  {
    message: "hello"
  }
}
// 期待通りに { message: "hello" } が返らず、undefinedになる

```

→ **return文も同じ行に値を書くか、必ず[[基礎 Part 5セミコロン・パース|セミコロン]]を！**

### ● まとめ指針

- **文の区切りには必ず[[基礎 Part 5セミコロン・パース|セミコロン]]をつける癖を！**
- プリティア([[Prettier]])など自動整形ツール導入で迷いを消そう

## [[基礎 Part 5セミコロン・パース|パース]]([[基礎 Part 5セミコロン・パース|parse]])とは？

- **プログラムの文字列をコンピュータが「意味のある構造」に分解・解釈すること**
- 日本語で言うと、文章を「主語」「動詞」「目的語」…に分けて意味を認識するイメージ
- [[基礎 Part 5セミコロン・パース|パース]]時点で意味不明な構文は「構文エラー」になる

## JSの実行プロセス（ざっくり）

1. **ソースコードを書く**
2. **[[基礎 Part 5セミコロン・パース|パース]]（構文解析）**
3. **抽象構文木(AST)を作る**
4. **バイトコードや中間コードに変換**
5. **JITなどで一部をネイティブコード化**
6. **実行**
- [[基礎 Part 5セミコロン・パース|パース]]は「ネイティブコード化」の前段階であり、“意味付け・下ごしらえ”の工程！
- [[基礎 Part 5セミコロン・パース|セミコロン]]は「[[基礎 Part 5セミコロン・パース|パース]]時に区切りを明示」するために必要

## よくあるQ&A

### Q. [[基礎 Part 5セミコロン・パース|セミコロン]]全部付けた方が良い？

A. → 迷ったら**全部付けてOK**。省略できても「バグ防止」のためクセにしよう！

### Q. return文は？

A. → **必ず同じ行に書くか、[[基礎 Part 5セミコロン・パース|セミコロン]]を！** （[[改行コード|改行]]だけだと罠になる）

### Q. オブジェクトには？

A. → **オブジェクト自体には不要**だが、「文として使うとき」はその文の終わりに付ける
